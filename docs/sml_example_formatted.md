# 機械学習モデルの評価レポート

## 1. 導入

本レポートでは、新たに開発した 予測モデル の性能評価結果について報告する。
評価にあたっては、以下の2つを主要な評価指標（メトリクス）として採用した。

- 予測精度 — F1スコアを基準とする
- 推論速度 — 推論1回あたりのレイテンシで測る

予測精度はモデルの実用性を担保するために最も重要な指標であり、今回はF1スコアを基準とする。

## 2. 評価結果（多次元表）

実験における各モデルの評価結果は以下の通りである。

**表: モデル別・データセット別の性能比較**

| model | Dataset-A / F1-Score | Dataset-A / Latency | Dataset-B / F1-Score | Dataset-B / Latency |
| --- | --- | --- | --- | --- |
| Baseline-v1 | 0.82 | 45 ms | 0.78 | 50 ms |
| Opt-v2 | 0.91 | 12 ms | 0.88 | 15 ms |

## 3. 分析と考察

評価結果の表（表: モデル別・データセット別の性能比較） から明らかなように、`Opt-v2` は `Baseline-v1` と比較して大幅な性能向上を達成している。

特に、Dataset-A における Opt-v2 のレイテンシ（セル: モデル別・データセット別の性能比較） (Opt-v2, Dataset-A.Latency) は **12 ms** であり、Baseline の 45 ms から約73%の高速化を実現している。行列積の並列パイプラインが効率的に機能しているためである。

推論の数学的ボトルネックは、以下の損失関数の計算部分であった。

$$
L = \frac{1}{N} \sum _{i = 1}^{N} ( y_{i} - {\hat{y}}_{i} )^{2}
$$

この 損失関数（数式: L = \frac{1}{N} \sum _{i = 1}^{N} ( y_{i} - {\hat{y}}_{i} )^…） の計算をGPU向けにカーネル最適化したことが、`Opt-v2` の速度向上の最大の要因である。
