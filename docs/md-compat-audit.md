# CommonMark/GFM 互換性監査(WP-M0、2026-07-15)

D39(SML は素の Markdown の上位互換)のための実測監査。全項目を最小入力で
`strata fmt` → `strata build` に通し、挙動を4分類した。検証入力・出力は
セッションのスクラッチにて(コード変更なし)。

分類: ①対応済み / ②**静かに壊れる**(無診断で情報が落ちる・化ける) /
③診断が出る / ④未対応(素通しでテキスト扱い・情報は残る)

## ① 対応済み

ATX 見出し(H1〜H6)/ 段落 / 強調 `**` `*` `` ` `` / ネストリスト /
基本リスト(`- `・`N. `)/ コードフェンス(```)/ フロントマターと文中 `---` の
無衝突(フロントマターはオフセット0限定判定)

## ② 静かに壊れる(最危険)

| # | 機能 | 入力例 | 実挙動 |
|---|---|---|---|
| 1 | エスケープ未実装 | `\*not emphasis\*` | `\` がリテラル残存した上で `*` が**本物の強調として誤発火**(emph ノード誤生成) |
| 2 | 外部リンク/画像の無診断不活性化 | `[Site](https://example.com)` | 丸ごと Text 化。Ref もエッジも生成されない(D39 名指しの既知違反)。`![alt](url)` も同様 |
| 3 | 画像+内部 ref の意味破壊 | `![alt](ref:target)` | `!` が孤立 Text として残り、残りが**通常の Ref として解決されエッジが張られる**(見せかけの正常動作) |
| 4 | 参照スタイルリンク | `[Example][ex]` + `[ex]: https://…` | 使用側は Text 化、**定義行が可視の段落ノードとして本文に出現** |
| 5 | 順序リストの開始値消失 | `5. fifth` | core/AST に `start` の概念自体が無く、再レンダーで 1 起番に化ける |
| 6 | ゆるいリストの分裂 | `- a`(空行)`- b` | 空行 = 無条件ブロック境界のため**項目数ぶんの独立 List ノード**に分裂。#5 と複合すると順序リストが全項目「1.」化しうる |
| 7 | `***bold italic***` 誤ネスト | 同左 | 貪欲 `**` マッチで `*` が strong 内にはみ出し、閉じ `*` が本文に漏れる |
| 8 | 未対応ブロック内のインライン誤爆 | `~~~` フェンス内の `x * 2 * y` | tilde フェンスが段落化された上で**中身の `*` が強調として二次誤パース**(コードが変形される) |
| 9 | Setext 見出しの構造喪失 | `Title\n=====` | 2行まるごと1段落。Section にならず文書階層に参加しない。Setext H2 の `---` は HR と同形の認知的罠 |

## ③ 診断が出る

- 未知スキームのリンク `[x](foo:bar)` → `UnknownScheme`(Error、全か無か)
- `mailto:` も同様に Error(CommonMark では日常的スキームだが SML の6スキーム外)

## ④ 未対応(素通し)

blockquote(`>` がリテラル残存)/ 水平線(空行有無で吸収先が変わる非対称あり)/
**GFM パイプ表**(1段落 Text 化。属性行との構文衝突は無し)/ タスクリスト
(`[ ]` がリテラル)/ 脚注(定義行可視化 — ②寄り)/ HTML ブロック・インライン /
ハードブレイク(バイト保存)/ リンクテキスト内の書式(`[**B**](ref:x)` の
text が生文字列)/ 見出し閉じ `#` 装飾(本文残存)/ 代替記法一式
(`*`・`+` 箇条書き、`1)`、`_em_`・`__strong__`、`~~~` フェンス、`~~取消線~~`)

## 固有記法との衝突・注意

- フロントマター vs 文中 `---`: 衝突なし(実測)
- GFM 表 vs 属性行: 構文衝突なし。ただし単独行 `[ ]`(空チェックボックス)は
  属性行と誤認され `BadKeyCharset` になる可能性(ロジック上。未実測)
- 監査の総括: バイト保存の意味での round-trip は④でほぼ成立しているが、
  「意味・構造・機能の保存」では②の9項目が D39 に反する。優先度は
  ②1(エスケープ)・②8(インライン誤爆)・②5+6(リスト)・②2(外部リンク)が上位
